# Заметки по работе сети 


### 1. Локальный запуск (через терминал poetry run)
При запуске сервисов просто через `poetry run uvicorn src.app.main:app --host --port`, они работают в одной системе. В этом случае они спокойно общаются через `127.0.0.1:{порт}`. Мой компьютер здесь выступает как общая площадка для всех процессов.

### 2. Запуск в контейнере через 127.0.0.1 (Почему это ломается)
Если я засуну `order-service` в контейнер и оставлю ему адрес `127.0.0.1`, всё сломается. 
**Причина:** для контейнера `127.0.0.1` — это он сам. Он пытается найти сервис скидок или товаров внутри себя, но их там нет (они в других контейнерах).

### 3. Успешный запуск через имена сервисов (Docker)
В `docker-compose.yml` мы объединили всё в одну сеть. Пишем `http://product-service:8000`. 
Docker сам  видит имя `product-service`, понимает, какому контейнеру оно принадлежит, и соединяет их. 

### 4. Использование host.docker.internal
Этот адрес пригодился бы, если бы, например, база данных Postgres была установлена локально(вне Докера). Тогда контейнеры стучались бы к ней не по имени сервиса, а через этот специальный хост.
